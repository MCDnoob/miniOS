# ELF文件格式

Executable and Linking Format / 可执行和链接的格式

1. 可执行程序，通过 python / bash / gcc 生成的文件
2. 可重定位文件，通过 gcc - c生成的文件，结尾是`.o` / 静态库，通过 ar生成的文件，结尾是`.a`，两者在内容上应该没什么区别，静态库链接可能会更快一些
3. 共享的目标文件，即动态链接库，`.so`结尾。

## 可执行程序

ELF可执程序中，有两个关键词：
1. section 这是一个ELF概念；
2. segment 这是一个CPU概念；
简单的可以理解为，一个segment由多个section组成。

1. 代码 .text段
2. 数据：
   1. .data section 已经初始化过的数据
   2. .bss section 未初始化过的数据
